#### while循环
```c
short index = 1
while( --index < 5 )
```
>当index减少到正数  32767 short有符号存储的最大值时，会停止循环

#### 真值问题
>scanf（“%d”,&num）函数，如果读取的不是num对应数据类型，会返回0
如果在循环中（非循环也一样），scanf（）读取非对应数据类型，会将读取错误的值放在队列下次读取，用户无法介入，会导致死循环。

#### for循环的执行顺序
```c
for (int i = 36; i > 0; i/=2)   // i/=2是在循环体循环结束后才会执行。
{	
	printf(" %d", i);
}
```

#### 常见的坑
- 自增/自减的前置与后置差别
>i = ++j; 与 i = j++; 不同（前者先加再用，后者先用再加）。避免在同一个表达式里混用 i++ 和对 i 的其他写操作（未定义行为）。

- 赋值 = 与相等比较 == 混淆
>如 a & b == 0 实际为 a & (b == 0)，可用括号明确意图：(a & b) == 0。

- 有符号与无符号比较问题
>比较 int 与 unsigned 时会把 int 转为 unsigned，负数会变成很大值。保持类型一致或显式转换。

- 格式化字符串与类型不匹配
>printf("%d", long_value); 在不同平台导致错误。使用正确的格式（如 "%ld"）或 <inttypes.h> 中的宏。

- 缓冲区与字符串终止符
>char s[3] = "ab"; OK, 但 char t[2] = "ab"; 会溢出。读字符串用长度限制：scanf("%2s", buf)（或更安全的 fgets）。

- sizeof 与字符串长度不同
>sizeof(arr) 是数组在编译时的字节大小，strlen(arr) 是到 '\0' 的长度。对于传入函数的数组参数，常退化为指针，sizeof 不再给元素数。


- 浮点比较不要直接用 
>\==用差的绝对值小于容差：fabs(a - b) < 1e-6。另外浮点运算有舍入误差。

- 顺序点/未定义行为（更高级）
>表达式中对同一对象多次写且无序列点（如 i = i++ + 1;）是未定义行为，应拆分成多条语句。

- 指针和数组常见误用
>不要对非数组的指针做 sizeof 获取元素数；解引用空指针或越界访问会导致未定义行为或崩溃。

- scanf 的返回值与输入验证
>scanf 返回成功转换的字段数，检查返回值避免使用未初始化的变量：if (scanf("%d", &n) == 1) ...
