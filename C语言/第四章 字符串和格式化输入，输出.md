#### 类型可移植性
>sizeof运算符标准定义是无符号，实际上可以是unsigned int ... 因此，入股哦使用printf函数显示sizeof表达式，根据不同系统还需要切换转换说明。C提供了可移植性更好的类型。stddef.h头文件（包含在stdio.h）吧size_t定义成系统使用sizeof返回的类型，这被称之为底层类型(underlying type).其次，printf使用z修饰符打印相应类型。同样C还定义了ptrdiff_t类型和t修饰符来表示系统使用的两个地址差值的底层有符号整数类型。

#### 参数传递机制
```c
printf("%ld %ld %ld %ld \n",n1,n2,n3,n4)
```
>程序将传递过来的**参数根据变量类型放入栈中**。n1 n2 8字节。n3 n4 4字节。然后控制转到printf函数。**printf根据转换说明类型长度读取栈中参数。**%ld读四个字节，所以前四个字节作为n1.这是n1的前半部分，却被解释成了一个整数。%ld在读取下一个字节，n1后半部分，被解释成n2。

#### printf scanf \*用法
```c

printf("%4d",a) //a是123则会输出 1个空格123  在两个转换说明中添加一个空格，就算输出溢出，两个转换说明中间始终有一个空格，这样就会看上去不像是一个值。
```

#### scanf用法注意
%c缓冲区设置大于1的情况
当我们需要依次输入多个字符时候，使用%nc，缓冲区设置n，末尾加\0
```c
char code[5];   // 4 个字符 + 空字符位置
scanf_s("%4c", code, 4);   // 读取 4 个字符，不自动加 '\0'

code ['a'] ['b'] ['c'] ['d'] ['\0']

```
错误行为,必须要提供参数，否则是未定义的行为
```c
char ch;

```

#### 关于 &str &ch
有时候会看到&str这种写法，虽然str数组名已经是指针，但是&str取的是整个数组的地址。类型是
```c
str类型 字符串   &str类型 char (*)[10]
&ch 类型 char(*)[10]  包含了10个char的数组的指针
```
在数值上与str相同，但是类型不同，而且可能会引起编译器报警(类型不匹配) 不推荐
